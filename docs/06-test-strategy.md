# 테스트 전략

## 1. 테스트 목표

- 게시글 작성부터 발행 결과 저장까지의 핵심 흐름을 검증한다.
- 외부 API 장애와 인증 만료를 안정적으로 처리한다.
- 예약 발행이 중복 실행되지 않음을 보장한다.
- 광고/개인정보 정책 위반 가능 기능을 사전에 차단한다.

## 2. 테스트 범위

### 단위 테스트

- 텍스트 길이 검증
- 이미지 규격 검증
- 플랫폼별 콘텐츠 변환
- 재시도 정책
- 오류 코드 정규화
- 권한 검사

### 통합 테스트

- 게시글 생성 후 PublishJob 생성
- Worker가 Job을 가져와 Adapter 실행
- 성공 결과 저장
- 실패 결과 저장
- 실패 플랫폼만 재시도
- 토큰 만료 시 계정 상태 변경

### E2E 테스트

- 로그인
- 게시글 작성
- 이미지 업로드
- 플랫폼 선택
- 즉시 발행
- 예약 발행
- 발행 결과 확인

## 3. 외부 API 테스트 전략

실제 API 호출 테스트와 Mock 테스트를 분리한다.

- Mock Adapter: CI에서 항상 실행
- Sandbox 또는 테스트 계정: 스테이징에서 실행
- 실제 계정 발행: 수동 승인 후 제한적으로 실행

## 4. 주요 시나리오

- 정상 게시글 즉시 발행
- 예약 시간이 된 게시글 발행
- 한 플랫폼은 성공하고 다른 플랫폼은 실패
- API rate limit 발생
- 토큰 만료
- 이미지 용량 초과
- 플랫폼별 텍스트 길이 초과
- 사용 권한 없는 계정으로 발행 시도
- 동일 예약 작업 중복 실행 방지

## 5. 품질 게이트

- 핵심 단위 테스트 통과
- Worker 통합 테스트 통과
- 민감 정보 로그 미출력 확인
- 실패 재시도 정책 검증
- 정책상 금지 기능 미포함 확인
