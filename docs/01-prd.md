# 제품 요구사항 정의서

## 1. 목적

사용자가 하나의 작성 화면에서 텍스트와 이미지를 구성하고, 선택한 플랫폼에 맞게 내용을 조정한 뒤 Instagram, Threads, Kakao 계열 채널에 즉시 또는 예약 발행할 수 있는 자동 포스팅 시스템을 구축한다.

## 2. 성공 기준

- 사용자는 하나의 게시글을 작성해 여러 플랫폼에 발행할 수 있다.
- 플랫폼별 발행 성공/실패 상태를 확인할 수 있다.
- 실패한 플랫폼만 재시도할 수 있다.
- 예약 발행이 중복 실행되지 않는다.
- OAuth 토큰과 API 키는 안전하게 저장된다.
- Kakao 발송은 공식 API와 광고/개인정보 정책을 준수한다.

## 3. 사용자

- 관리자: 사용자, 조직, 소셜 계정, 발행 로그를 관리한다.
- 마케터: 게시글을 작성하고 예약 발행한다.
- 승인자: 운영 조직이 필요로 할 경우 발행 전 게시글을 승인한다.
- 조회자: 발행 이력과 결과를 확인한다.

## 4. 기능 요구사항

### 4.1 게시글 작성

- 원본 텍스트 입력
- 이미지 업로드
- 플랫폼별 텍스트 개별 수정
- 해시태그 입력
- 링크 입력
- 초안 저장
- 게시글 복제
- 플랫폼별 미리보기

### 4.2 플랫폼 선택

- Instagram
- Threads
- Kakao 공식 발송 채널
- 플랫폼별 계정 연결 상태 표시
- 플랫폼별 텍스트 길이와 이미지 제약 검증
- API 권한 상태 표시

### 4.3 발행

- 즉시 발행
- 예약 발행
- 선택 플랫폼 발행
- 실패 플랫폼만 재시도
- 발행 취소
- 발행 상태 조회

### 4.4 계정 연동

- OAuth 기반 계정 연결
- 토큰 만료 감지
- 토큰 갱신
- 계정별 발행 권한 관리
- 여러 계정 관리

### 4.5 결과 관리

- 플랫폼별 성공/실패 상태 저장
- 플랫폼별 게시 URL 저장
- 실패 사유 저장
- API 오류 코드 저장
- 관리자용 감사 로그 저장

## 5. 비기능 요구사항

### 5.1 보안

- OAuth 토큰 암호화 저장
- API 키 환경변수 관리
- 민감 정보 로그 저장 금지
- 역할 기반 접근 제어
- 관리자 작업 감사 로그 저장

### 5.2 안정성

- 큐 기반 비동기 발행
- 예약 작업 중복 실행 방지
- 플랫폼 장애가 다른 플랫폼 발행에 영향을 주지 않도록 격리
- 재시도 정책 적용
- 장애 알림

### 5.3 확장성

- 플랫폼별 어댑터 구조
- 새 SNS 플랫폼 추가 가능
- 이미지 처리 모듈 분리
- 예약/발행 엔진 분리

## 6. MVP 범위

### 포함

- 로그인
- 조직과 사용자 기본 관리
- 게시글 작성
- 이미지 1장 업로드
- Instagram 게시
- Threads 게시
- Kakao 공식 발송 방식 1개 확정 후 연동
- 즉시 발행
- 예약 발행
- 실패 재시도
- 발행 로그

### 제외

- AI 문구 생성
- 고급 성과 분석
- 댓글/DM 관리
- 대량 계정 운영
- A/B 테스트
- 승인 워크플로우 고도화

## 7. 1차 범위 축소 결정

초기 버전은 운영자 본인이 소유하거나 명확한 운영 권한을 가진 계정만 연결하는 내부용 도구로 진행한다.

포함:

- 운영자 본인의 Instagram Business 또는 Creator 계정
- 운영자 본인의 Threads 계정
- 운영자 본인의 Kakao 공식 채널 또는 비즈메시지 발송 계정
- 단일 운영자 또는 소규모 내부 팀 사용자
- 공식 OAuth/API 기반 게시

제외:

- 외부 고객용 SaaS
- 타인 계정 대행 운영
- 결제/과금
- 다수 고객 계정 관리
- 플랫폼 비밀번호 또는 로그인 세션 저장
