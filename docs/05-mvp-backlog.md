# MVP 개발 백로그

## Phase 0. 정책/권한 검증

- 전문가 TF 구성
- 로그인/OAuth 기반 자동화 경계 확정
- Kakao 공식 발송 방식 확정
- Instagram Business/Creator 계정 준비
- Meta App 생성과 권한 확인
- Threads API 사용 가능 범위 확인
- 광고성 메시지 여부 확정

완료 기준:

- 각 플랫폼별 공식 구현 경로가 문서화된다.
- MVP에서 제외할 위험 구현 방식이 확정된다.

## Phase 1. 프로젝트 기반

- 저장소 초기화
- Backend API 프로젝트 생성
- Frontend 프로젝트 생성
- PostgreSQL 연결
- Redis Queue 연결
- 기본 사용자/조직 모델 생성

완료 기준:

- 로컬에서 API와 Web App이 실행된다.
- 기본 헬스체크가 통과한다.

## Phase 2. 게시글 작성

- 게시글 생성/수정/조회 API
- 이미지 업로드 API
- 플랫폼별 콘텐츠 저장
- 작성 화면
- 플랫폼별 검증 메시지

완료 기준:

- 사용자가 게시글과 이미지 1장을 저장할 수 있다.
- 플랫폼별 문구를 별도로 저장할 수 있다.

## Phase 3. 계정 연동

- SocialAccount 모델
- OAuth 연결 흐름
- 토큰 암호화 저장
- 계정 상태 표시

완료 기준:

- Instagram 또는 Threads 계정을 연결하고 상태를 확인할 수 있다.

## Phase 4. 발행 엔진

- PublishJob 생성
- 즉시 발행 큐 처리
- 예약 발행 큐 처리
- 실패 재시도
- PublishResult 저장

완료 기준:

- 테스트 어댑터로 성공/실패/재시도 흐름을 검증할 수 있다.

## Phase 5. 플랫폼 어댑터

- Instagram Adapter
- Threads Adapter
- Kakao Adapter
- 오류 코드 정규화
- API 제한 오류 처리

완료 기준:

- 최소 1개 실제 플랫폼에 게시글을 발행할 수 있다.

## Phase 6. 관리자/운영 화면

- 계정 목록
- 발행 이력
- 실패 사유
- 재시도 버튼
- 감사 로그 조회

완료 기준:

- 운영자가 발행 상태와 실패 원인을 확인할 수 있다.

## Phase 7. QA와 배포 준비

- API 단위 테스트
- Worker 통합 테스트
- 예약 발행 테스트
- 보안 점검
- 배포 파이프라인
- 모니터링

완료 기준:

- MVP 릴리스 후보가 스테이징 환경에서 검증된다.

## 범위 축소 결정

MVP는 운영자 본인 계정 전용 내부 도구로 진행한다.

추가 완료 기준:

- 외부 고객용 SaaS, 대행 운영, 다중 고객 계정 관리는 MVP에서 제외된다.
- 단일 운영자 계정 기준 권한 모델을 우선 구현한다.
- 플랫폼 비밀번호와 로그인 세션 쿠키를 저장하지 않는다.
- 공식 OAuth/API 기반 자동화만 구현한다.
