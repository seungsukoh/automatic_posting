# 리스크 관리표

## 1. Kakao 일반 채팅방 자동 발송 불가

- 영향도: 높음
- 가능성: 높음
- 대응: KakaoTalk Channel, Business Message, Alimtalk, Friendtalk 등 공식 발송 방식으로 범위 전환

## 2. Meta API 권한 심사 지연

- 영향도: 높음
- 가능성: 중간
- 대응: 테스트 앱과 테스트 계정으로 PoC 선행, 실제 릴리스 일정에는 심사 버퍼 반영

## 3. Threads API 기능 제한

- 영향도: 중간
- 가능성: 중간
- 대응: MVP에서 텍스트/이미지 단일 게시부터 시작하고 제한 기능은 후순위화

## 4. 광고성 메시지 규제 위반

- 영향도: 높음
- 가능성: 중간
- 대응: 수신 동의, 광고 표기, 수신 거부, 야간 발송 제한을 요구사항에 포함

## 5. OAuth 토큰 유출

- 영향도: 높음
- 가능성: 낮음
- 대응: 암호화 저장, secret manager, 로그 마스킹, 최소 권한 scope 적용

## 6. 예약 발행 중복 실행

- 영향도: 높음
- 가능성: 중간
- 대응: Job lock, idempotency key, DB transaction, 중복 실행 테스트 적용

## 7. 이미지 규격 오류로 발행 실패

- 영향도: 중간
- 가능성: 높음
- 대응: 발행 전 플랫폼별 이미지 검증과 자동 리사이징 도입

## 8. 플랫폼 API 변경

- 영향도: 중간
- 가능성: 중간
- 대응: Adapter 구조, API 버전 관리, 플랫폼별 모니터링 적용

## 9. 외부 고객용 SaaS 전환 시 요구사항 증가

- 영향도: 높음
- 가능성: 중간
- 대응: 내부용 MVP와 SaaS용 기능을 명확히 분리

## 10. 개인 운영 계정 범위 초과

- 영향도: 중간
- 가능성: 중간
- 대응: MVP에서는 운영자 본인 소유 또는 운영 권한 보유 계정만 연결하도록 제품 정책과 UI 문구에 명시
