# 로그인과 자동화 가능 범위

## 1. 결론

로그인을 할 수 있으면 자동화 가능한 부분이 있다. 다만 안정적으로 제품화할 수 있는 범위는 "공식 로그인/OAuth로 권한을 받은 뒤 공식 API가 허용하는 작업"이다.

반대로 사용자가 로그인한 브라우저나 PC 앱 화면을 자동으로 조작해서 게시하거나 메시지를 보내는 방식은 MVP 범위에서 제외한다. 이 방식은 플랫폼 약관 위반, 계정 제재, UI 변경으로 인한 장애, 보안 사고 위험이 크다.

## 2. 로그인 유형별 가능 범위

### 2.1 우리 서비스 로그인

사용자가 Automatic Posting 서비스에 로그인하는 것이다.

가능한 기능:

- 사용자 식별
- 조직/팀 관리
- 게시글 작성
- 예약 발행 관리
- 발행 이력 조회
- 권한 관리
- 감사 로그 기록

이 로그인만으로 Instagram, Threads, Kakao에 게시할 수는 없다. 외부 플랫폼 계정 권한이 별도로 필요하다.

### 2.2 플랫폼 OAuth 로그인

사용자가 Instagram, Threads, Kakao 등 외부 플랫폼의 공식 인증 화면에서 권한을 허용하는 것이다.

가능한 기능:

- 연결된 계정 목록 조회
- 플랫폼이 허용한 범위의 게시글 발행
- 토큰 만료 전 갱신
- 발행 결과 조회
- 권한 부족 또는 인증 만료 감지

MVP에서 목표로 삼아야 할 방식이다.

### 2.3 로그인된 화면 자동 조작

브라우저나 PC 앱에 로그인된 상태를 이용해 버튼 클릭, 파일 첨부, 메시지 전송을 자동으로 조작하는 방식이다.

MVP 제외 사유:

- 공식 API가 아닌 우회 구현
- 플랫폼 UI 변경 시 즉시 장애 발생
- 자동화 탐지와 계정 제재 위험
- 사용자 세션 탈취/보관 문제
- 대량 발송 또는 광고 발송 시 법적 리스크 증가
- 장애 원인 추적과 고객지원이 어려움

## 3. 플랫폼별 판단

### 3.1 Instagram

가능성이 높다. 단, 일반 개인 계정이 아니라 Business 또는 Creator 계정과 Meta Graph API를 기준으로 해야 한다.

필요한 것:

- Meta App
- Instagram Business/Creator 계정
- 필요한 API 권한
- OAuth 연동
- 게시 API PoC

### 3.2 Threads

공식 API 가능 범위를 확인해야 한다. 텍스트와 이미지 게시가 API로 허용되면 OAuth 기반 자동화가 가능하다.

필요한 것:

- Threads API 권한 확인
- OAuth 또는 Meta 계정 연동 방식 확인
- 게시 API PoC

### 3.3 Kakao

가장 신중해야 한다. KakaoTalk 일반 개인/단체 채팅방 자동 발송은 로그인되어 있어도 제품화 대상으로 보기 어렵다.

가능성이 있는 공식 경로:

- KakaoTalk Channel
- Kakao Business Message
- Alimtalk
- Friendtalk

필요한 것:

- 발송 대상 정의
- 광고성 메시지 여부
- 수신 동의 관리
- 수신 거부 처리
- 야간 발송 제한 검토
- Kakao 공식 API 또는 비즈메시지 계약 검토

## 4. PoC 우선순위

1. 우리 서비스 로그인과 사용자/조직 모델 구현
2. Instagram OAuth 연결 PoC
3. Instagram 테스트 게시 PoC
4. Threads OAuth/API 가능성 검증
5. Kakao 공식 발송 방식 결정
6. Kakao 공식 발송 PoC

## 5. 제품 정책

MVP 제품 정책은 다음과 같이 정한다.

- 공식 API 기반 자동화만 지원한다.
- 사용자의 플랫폼 비밀번호를 저장하지 않는다.
- 외부 플랫폼 로그인 세션 쿠키를 저장하지 않는다.
- 화면 조작 기반 자동 발송은 지원하지 않는다.
- 발행 권한은 OAuth scope 기준으로 최소화한다.
- 모든 발행 작업은 감사 로그를 남긴다.

## 6. 1차 제품 범위

초기 범위는 운영자 본인이 소유하거나 명확한 운영 권한을 가진 계정만 연결하는 내부용 도구로 제한한다.

포함:

- 운영자 본인 Instagram 계정 OAuth 연결
- 운영자 본인 Threads 계정 OAuth/API 연결
- 운영자 본인 Kakao 공식 채널 또는 비즈메시지 계정 연결
- 게시글 작성, 이미지 첨부, 즉시 발행, 예약 발행

제외:

- 외부 고객 계정 위임
- 대행사형 다중 계정 운영
- 결제/과금
- 플랫폼 비밀번호 저장
- 로그인 세션 쿠키 저장
- 화면 조작 기반 자동 발송
